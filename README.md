# argocd-manifests

## 说明
- 使用argocd部署容器，学习jsonnet语言。

- 将之前的以`区域和集群目录`规划改为`区域和集群代码分支`模式。

  - 原目录结构
  ```shell
  -- argocd-manifests
    -- bj
      -- k8s-cluster-prod-001
          -- consul-agent
          -- consul-server
          -- ...
          -- ...
          -- ...
          -- ...
    -- gd
      -- k8s-cluster-prod-001
          -- ...
          -- ...
          -- ...
          -- ...
    -- hk
      -- k8s-cluster-prod-001
          -- ...
          -- ...
          -- ...
          -- ...
    -- sh
      -- k8s-cluster-prod-001
          -- ...
          -- ...
          -- ...
          -- ...
  ```

  - 现分支模式
    - master
    - bj-k8s-cluster-prod-001
    - gd-k8s-cluster-prod-001
    - hk-k8s-cluster-prod-001
    - sh-k8s-cluster-prod-001

## 
- 语法格式
    - 基础建设类的就使用plain yaml文件实现，这一类的服务部署一套就可以了，如prometheus-operator。
    - 中间件类的服务使用jsonnet实现，毕竟这一类的服务就涉及到部署多套，如果jenkins服务，不同的用户使用不同的jenkins。

- 目录说明
  - `__yaml`，存放plain yaml文件，不用于jsonnet渲染。
  - `out-of-the-box`，开箱即用
